Purpose

This directory contains the examples and applications to demonstrate the capabilities of the P-NUCLEO-WB55 Kit.

Most of those projects are generated with STM32CubeMX tool to initialize the system, peripherals and middleware stacks.

Several applications are provided ready to use with IAR©, KEIL© and STM32CubeIDE© to demonstrate the Bluetooth Low Energy, Thread and Mac 802-15-4 capabilities of the device.

A particular attention must be taken when using BLE, Thread or Mac 802-15-4 as a specific binary must be loaded inside device memory for each application. You can refer to the readme.txt of each application for the detailed information and usage.

They can be found under the following directories:

  • Projects\NUCLEO-WB55.USBDongle\Applications\BLE
  • Projects\NUCLEO-WB55.USBDongle\Applications\Thread
  • Projects\NUCLEO-WB55.USBDongle\Applications\Zigbee

Maintenance

  • ID 178250 : USB Device ID are now based on UID register to have a unique identifier by STM32 chip.

  • BLE updates :
    • ID 178246 : BLE characteristics greater than 255 bytes are now managed properly in CubeMX interface and generated code.
  • Thread updates :
    • ID 182875 : Increased Thread RCP stability, robustness and efficiency:
      • Spinel over UART interface with Host is now set on ST-LINK’s USB (no more USB/serial converter cable needed).
      • Spinel over UART interface is set to a higher priority than debug traces, which are now disabled by default.
      • Enabled FIFO mode and DMA on UART/LPUART.
      • UART’s baudrate is now set to 460800, traces (if enabled) are still set to 115200.
      • Disabled low-power.

Maintenance

  • Update of FW projects following changes in latest version of HAL/LL and Middlewares.

  • ZIGBEE
    • ID 156968 : When the Zigbee startup fails, the memory previously allocated was now correctly freed. This was impacting all Zigbee applications running on STM32WB5MM-DK, on P-NUCLEO-WB55.USBDongle and also the BLE_Zigbee dynamic mode application as well.
    • ID 163800 : Correct an issue when displaying the temperature measurement on the Zigbee TempMeas application running on STM32WB5MM-DK using the CubeIDE toolchain.
    • ID 163526 : Setup of the correct priority associated to the button management used on the Zigbee_OnOff_Client_Router application running under ThreadX.
    • ID 160728 : Rework of the APP_ZIGBEE_ProcessNotifyM0ToM4() and APP_ZIGBEE_ProcessNotifyM0ToM4() functions used on all Zigbee applications to handle IPPC mechanism. The purpose of these updates is to avoid potential deadlocks in very extreme conditions (very noisy radio environment with multiple messages to be processed at the same time).
